# 智学工坊文档入口

本文档用于帮助开发者和 AI 编程助手快速定位项目设计资料。执行任务前应先阅读 `AGENTS.md`、`docs/Codex开发任务拆分.md` 和当前任务相关模块文档。

> 当前推进约束：教师端和管理员端新增建设已冻结。后续不要新增 `/teacher/*`、新的 `/admin/*` 页面或管理员演示初始化接口；优先修复并验证学生端主链路。

## 推荐阅读顺序

1. 项目定位：`02_项目定位与创新设计/`
2. PRD：`03_PRD产品需求文档/PRD.md`
3. 系统架构：`06_系统架构设计/06_系统架构设计.md`
4. 数据库：`10_数据库设计/数据库设计.md`
5. API：`11_API接口设计/API接口设计.md`
6. 前端页面：`12_前端页面设计/前端页面设计.md`
7. UI 规范：`13_UI设计规范/UI设计规范.md` 与 `DESIGN.md`
8. Agent、Wiki、自进化：`07_多智能体架构设计/`、`09_LLM_Wiki学习空间设计/`、`08_自进化学习智能体设计/`
9. 部署与测试：`20_部署方案/`、`19_测试方案/`

## 主要文档分区

- `00_文档规范/`：术语、命名和文档维护规则。
- `01_赛题解读/`：中国软件杯 A3 赛题要求和评审重点。
- `03_PRD产品需求文档/`：产品目标、需求优先级、验收标准。
- `05_功能模块设计/`：课程、资料、Wiki、问答、诊断、推荐等模块设计。
- `06_系统架构设计/`：前后端分层、容器、数据和异步任务架构。
- `10_数据库设计/`：表结构、ER、索引、迁移和种子数据要求。
- `11_API接口设计/`：统一响应、鉴权、业务 API 设计。
- `12_前端页面设计/`：路由、页面组件、状态和 API 调用方式。
- `13_UI设计规范/`：视觉规范；当前实现优先参考 `DESIGN.md` 的 Luminous Warmth。
- `14_大模型提示词工程设计/`：Prompt 分类、版本和质量校验。
- `15_RAG与知识库设计/`：资料解析、切片、Embedding 和检索。
- `16_学生画像与长期记忆设计/`：画像、偏好、长期记忆和隐私。
- `07_多智能体架构设计/`：Agent 角色、输入输出、日志和失败降级。
- `08_自进化学习智能体设计/`：策略版本、审批、回滚和日志。
- `09_LLM_Wiki学习空间设计/`：Wiki 页面、来源、版本和关系网络。
- `18_安全与隐私设计/`：认证、数据隔离、内容安全。
- `19_测试方案/`：单元、接口、前端、Agent、验收测试。
- `20_部署方案/`：本地、Docker、迁移、模型服务配置。
- `21_演示数据设计/`、`22_比赛材料规划/`、`23_演示视频脚本/`、`24_答辩稿/`：比赛演示与材料。

## 维护规则

- 新增功能前先确认任务编号和相关设计文档。
- 设计与实现冲突时优先遵守文档；确需调整时在任务输出中说明。
- 数据库结构变化必须同步模型、migration、Schema、API 和必要文档。
- AI、Wiki、自进化、Agent 相关实现必须保留来源、证据、版本和日志边界。
