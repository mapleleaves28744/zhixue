## 原则

当前阶段先把本地主链路打通，再集中处理 Docker 部署。日常开发默认使用本地 Python、Node、本机 PostgreSQL 和本机 Redis 服务。

## 本地环境变量

`.env` 默认应使用本机数据库和缓存：

```env
DATABASE_URL=postgresql+asyncpg://zhixue:zhixue_password@localhost:5432/zhixue
REDIS_URL=redis://localhost:6379/0
```

不要在本地开发 `.env` 中使用 Docker 内部主机名：

```env
DATABASE_URL=postgresql+asyncpg://zhixue:zhixue_password@postgres:5432/zhixue
REDIS_URL=redis://redis:6379/0
```

上面的 `postgres`、`redis` 只用于 Docker Compose 内部网络。

## 后端启动

```powershell
cd C:\Users\28744\Desktop\zhixue\backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m alembic upgrade head
uvicorn app.main:app --reload
```

健康检查：

```powershell
Invoke-RestMethod http://localhost:8000/health
```

## 前端启动

```powershell
cd C:\Users\28744\Desktop\zhixue\frontend
npm install
npm run dev
```

访问：

```text
http://localhost:3000
```

## 数据库与缓存

本地开发默认使用本机 PostgreSQL/Redis。请先确认：

1. PostgreSQL 正在监听 `localhost:5432`；
2. 已创建用户 `zhixue` 和数据库 `zhixue`；
3. Redis 正在监听 `localhost:6379`；
4. `python -m alembic upgrade head` 能连接并执行 migration。

Docker 中的 PostgreSQL/Redis 只作为部署阶段或临时备选，不作为日常开发默认依赖。

## LLM 配置

无真实 Key 时：

```env
LLM_PROVIDER=mock
EMBEDDING_PROVIDER=mock
```

接入 OpenAI-compatible 服务时：

```env
LLM_PROVIDER=compatible
LLM_BASE_URL=https://api.xiaomimimo.com/v1
LLM_API_KEY=你的_API_KEY
LLM_MODEL_NAME=mimo-v2.5-pro
EMBEDDING_PROVIDER=mock
```

## 验证顺序

1. 后端 `/health` 正常；
2. `python -m alembic upgrade head` 正常；
3. `python -m pytest` 或相关测试通过；
4. 前端页面可打开；
5. 完成注册、建课、资料上传、解析、切片、Wiki 生成等主链路验证。

Docker Compose 在部署阶段再作为完整验收项。
