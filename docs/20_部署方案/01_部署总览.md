## 部署优先级

当前阶段采用“本地优先，Docker 后置”的工程策略。

1. 功能开发和主链路打通优先使用本地开发环境，包括本机 PostgreSQL 和 Redis。
2. Docker Compose 用于部署专项、第21阶段和比赛演示环境整理。
3. 非部署任务不要因为 Docker 容器问题阻塞业务功能开发。
4. Docker 问题应记录为部署 TODO，等本地主链路稳定后集中处理。

## 环境分层

| 环境 | 用途 | 当前优先级 |
|---|---|---|
| 本地开发环境 | 日常开发、接口联调、LLM Provider 调试、本机数据库 migration 验证 | 最高 |
| 本地 Docker Compose | 后续部署模拟、演示环境、依赖统一 | 后置 |
| 服务器部署 | 比赛展示或远程访问 | 后置 |

## 当前验收原则

在主链路未完整稳定前，优先验收：

1. `python -m pytest` 或关键后端接口可运行；
2. `python -m alembic upgrade head` 可运行；
3. FastAPI 本地 `/health` 正常；
4. 前端本地开发服务可访问；
5. Mock Provider 和真实 OpenAI-compatible Provider 可切换。

Docker Compose 可启动仍是最终工程目标，但不作为当前每个业务任务的强制前置。

本地默认连接：

```env
DATABASE_URL=postgresql+asyncpg://zhixue:zhixue_password@localhost:5432/zhixue
REDIS_URL=redis://localhost:6379/0
```
