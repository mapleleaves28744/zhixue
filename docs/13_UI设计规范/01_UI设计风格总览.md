# 01_UI设计风格总览

> 当前视觉基准：`docs/DESIGN.md` 中的 **Luminous Warmth**。后续所有前端页面、组件、动效和数据可视化都应优先遵守该设计系统。

## 1. 当前视觉方向

智学工坊当前采用 Luminous Warmth 风格：

1. 温润浅色背景，而不是冷白后台风。
2. Glassmorphism 玻璃拟态卡片，强调透明、模糊、柔和边缘。
3. 主色为金棕与暖橙：`#835400`、`#f9a826`。
4. 支持色为低饱和粉、蓝灰、柔和紫灰。
5. 文本使用深炭色，避免纯黑造成生硬对比。
6. 页面应呈现“calm clarity”：安静、清晰、高级、可演示。

## 2. 字体基准

| 用途 | 字体 | 说明 |
|---|---|---|
| 标题 / 大数字 | Plus Jakarta Sans | 几何、现代、略圆润 |
| 正文 / 标签 | Manrope | 可读性高，适合学习场景 |
| 图标 | Material Symbols Outlined | 当前 Stitch 页面使用的图标体系 |

后续组件化时不应随意改用另一套字体或图标库，除非明确替换全局视觉系统。

## 3. 页面气质

页面应像一个温暖的个性化学习空间，而不是普通后台管理系统。

应突出：

1. 学习状态；
2. 知识组织；
3. AI 输出依据；
4. Agent 协作过程；
5. 自进化策略证据；
6. 推荐理由；
7. Wiki 来源追溯。

避免：

1. 大面积冷色纯白；
2. 黑色科技风；
3. 普通后台表格模板；
4. 随机渐变；
5. 廉价机器人插画；
6. 无解释、无来源的 AI 输出。

## 4. 当前实现注意事项

当前前端仍以 Stitch 静态 HTML 为视觉基准，Next.js 使用 iframe 承载。

后续重构要求：

1. 不改变 Stitch 页面已有外观、颜色、布局、圆角和密度。
2. 先抽取通用导航和玻璃卡片，再拆页面业务组件。
3. 每次替换静态 HTML 前后都需要浏览器对比。
4. 页面最终应迁移到 React + TypeScript + Tailwind 组件，但视觉必须与 Luminous Warmth 保持一致。
