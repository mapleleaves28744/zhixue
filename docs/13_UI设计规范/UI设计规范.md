# 智学工坊 UI 设计规范

文档版本：V1.0  
适用位置：`docs/UI设计规范.md`  
适用对象：前端开发、UI 设计、Codex 前端生成、页面验收、答辩展示  
技术栈建议：Next.js + TypeScript + Tailwind CSS + shadcn/ui + Framer Motion + Recharts  
项目定位：基于自进化学习智能体与 LLM Wiki 的个性化学习空间

---

# 1. 设计理念

## 1.1 核心定位

智学工坊不是普通后台管理系统，也不是单纯 AI 聊天工具，而是一个面向高校学生的**个人学习空间**。

视觉上应传达：

1. 高级；
2. 教育科技；
3. 温暖陪伴；
4. 知识空间；
5. 轻量未来感；
6. 柔和渐变；
7. 卡片式学习空间；
8. Wiki 知识网络；
9. 智能体协作感。

一句话设计原则：

> 用柔和、克制、结构化的教育科技视觉，呈现一个能持续理解学生、整理知识、生成资源、诊断学习状态的智能学习空间。

## 1.2 视觉气质

| 方向 | 要求 |
|---|---|
| 高级感 | 使用克制色彩、细腻阴影、统一间距、大圆角 |
| 教育感 | 内容层级清晰，知识结构可读，避免娱乐化 |
| 科技感 | 使用轻量渐变、知识网络、Agent 状态流、流式 AI 反馈 |
| 温暖感 | 避免冷冰冰黑客风，多使用浅色背景和柔和强调色 |
| 陪伴感 | 文案和界面反馈应像学习助手，而不是命令式后台 |
| 空间感 | 页面应像一个可探索的学习空间，卡片之间有层次和呼吸感 |

## 1.3 UI 总体原则

1. 页面应以学习任务和知识内容为中心，不以菜单和表格为中心。
2. 所有核心页面都应使用卡片化布局。
3. 背景应有轻微渐变和空间层次，但不能抢夺内容注意力。
4. 交互动效应轻微、自然，不应炫技。
5. AI 生成内容必须有来源、依据、状态或解释，不做“神秘黑盒”。
6. 页面应突出“为什么推荐”“为什么诊断”“为什么进化”。
7. 所有模块都应保持同一套设计语言。

# 2. 色彩规范

## 2.1 主色体系

推荐主色使用 **靛蓝 + 青蓝 + 翡翠绿**，体现智能、知识、成长。

| 色彩角色 | 色值 | 用途 |
|---|---|---|
| Primary Indigo | `#4F46E5` | 主要按钮、导航高亮、核心操作 |
| Primary Deep | `#3730A3` | 深色主按钮 hover、重点标题 |
| Cyan Tech | `#06B6D4` | AI 状态、知识网络、辅助高亮 |
| Emerald Growth | `#10B981` | 掌握度提升、完成状态、学习成长 |
| Amber Warning | `#F59E0B` | 待复习、注意提示、风险中等 |
| Red Error | `#EF4444` | 错误、薄弱严重、失败状态 |
| Slate Text | `#0F172A` | 主要文本 |
| Slate Muted | `#64748B` | 次级文本 |
| Slate Border | `#E2E8F0` | 边框 |
| Page Background | `#F8FAFC` | 页面背景 |

## 2.2 功能色规范

| 状态 | 背景 | 文本 | 边框 |
|---|---|---|---|
| 成功 / 已掌握 | `#ECFDF5` | `#047857` | `#A7F3D0` |
| 提醒 / 待复习 | `#FFFBEB` | `#B45309` | `#FDE68A` |
| 错误 / 薄弱 | `#FEF2F2` | `#B91C1C` | `#FECACA` |
| AI 处理中 | `#EEF2FF` | `#4338CA` | `#C7D2FE` |
| 知识关系 | `#ECFEFF` | `#0E7490` | `#A5F3FC` |
| 中性信息 | `#F1F5F9` | `#475569` | `#CBD5E1` |

## 2.3 渐变色规范

只允许使用柔和渐变，不使用高饱和霓虹渐变。

### 主视觉渐变

```css
background: linear-gradient(135deg, #4F46E5 0%, #06B6D4 50%, #10B981 100%);
```

使用位置：

1. 首页 Hero 标题局部；
2. 学生首页核心卡片；
3. AI 生成按钮；
4. 自进化总览卡片。

### 页面背景渐变

```css
background:
  radial-gradient(circle at top left, rgba(79, 70, 229, 0.10), transparent 32%),
  radial-gradient(circle at top right, rgba(6, 182, 212, 0.10), transparent 30%),
  linear-gradient(180deg, #F8FAFC 0%, #EEF2FF 100%);
```

使用位置：

1. 学生端整体背景；
2. 登录注册页；
3. Wiki 学习空间；
4. 自进化中心。

## 2.4 禁止色彩

禁止使用：

1. 大面积纯黑背景；
2. 高饱和荧光绿；
3. 大面积纯红；
4. 彩虹渐变；
5. 土味紫粉渐变；
6. 过多品牌色混用；
7. 随机颜色标签。

# 3. 字体规范

## 3.1 字体族

推荐：

```css
font-family: Inter, "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

| 场景 | 字体要求 |
|---|---|
| 中文正文 | PingFang SC / Microsoft YaHei / Noto Sans SC |
| 英文和数字 | Inter |
| 代码内容 | JetBrains Mono / Menlo / Consolas |
| 图表标签 | Inter + 中文系统字体 |

## 3.2 字号层级

| 层级 | Tailwind | 用途 |
|---|---|---|
| Display | `text-4xl` / `text-5xl` | 首页 Hero 标题 |
| H1 | `text-3xl` | 页面主标题 |
| H2 | `text-2xl` | 页面分区标题 |
| H3 | `text-xl` | 卡片标题 |
| Body | `text-base` | 正文 |
| Small | `text-sm` | 辅助说明 |
| Tiny | `text-xs` | 标签、时间、状态 |

## 3.3 字重规范

| 场景 | 字重 |
|---|---|
| 页面主标题 | `font-bold` |
| 卡片标题 | `font-semibold` |
| 正文 | `font-normal` |
| 按钮 | `font-medium` |
| 标签 | `font-medium` |
| 图表说明 | `font-normal` |

## 3.4 行高规范

| 场景 | Tailwind |
|---|---|
| 标题 | `leading-tight` |
| 正文 | `leading-7` |
| AI 回答 | `leading-8` |
| 表格 | `leading-6` |

# 4. 背景规范

## 4.1 全局背景

学生端页面应使用浅色、柔和、具有空间感的背景。

推荐背景：

```tsx
<div className="min-h-screen bg-slate-50 bg-[radial-gradient(circle_at_top_left,rgba(79,70,229,0.10),transparent_32%),radial-gradient(circle_at_top_right,rgba(6,182,212,0.10),transparent_30%)]" />
```

## 4.2 知识空间纹理

可以使用非常轻的网格纹理，但透明度必须低。

```css
background-image:
  linear-gradient(rgba(15,23,42,0.03) 1px, transparent 1px),
  linear-gradient(90deg, rgba(15,23,42,0.03) 1px, transparent 1px);
background-size: 32px 32px;
```

使用位置：

1. Wiki 图谱页面；
2. 学习路径页面；
3. 首页 Hero 区；
4. 自进化中心顶部区域。

## 4.3 背景使用限制

禁止：

1. 大面积纯白无层次；
2. 大面积深色黑背景；
3. 背景过度复杂；
4. 使用明显 AI 机器人图案铺底；
5. 使用廉价科技蓝线条背景；
6. 使用过多粒子动效。

# 5. 卡片规范

## 5.1 基础卡片

所有核心内容应采用卡片式布局。

推荐 class：

```tsx
className="rounded-2xl border border-slate-200/70 bg-white/80 p-6 shadow-sm backdrop-blur-md"
```

## 5.2 重点卡片

用于学生首页总览、自进化洞察、诊断摘要等重点区域。

```tsx
className="rounded-3xl border border-white/70 bg-gradient-to-br from-white/90 to-indigo-50/80 p-6 shadow-md backdrop-blur-xl"
```

## 5.3 卡片层级

| 类型 | 用途 | 样式 |
|---|---|---|
| Level 1 | 页面主卡片 | 大圆角、轻阴影、较大内边距 |
| Level 2 | 内容分组卡片 | 中圆角、浅边框 |
| Level 3 | 信息标签卡片 | 小圆角、浅背景 |
| Highlight | 推荐、洞察、策略更新 | 渐变背景、图标、状态标签 |

## 5.4 卡片间距

| 场景 | 间距 |
|---|---|
| 页面分区 | `gap-6` / `space-y-6` |
| 卡片内部 | `space-y-4` |
| 信息组 | `gap-3` |
| 标签组 | `gap-2` |

## 5.5 卡片动效

允许：

```tsx
whileHover={{ y: -2, scale: 1.005 }}
transition={{ duration: 0.18 }}
```

禁止：

1. 大幅度跳动；
2. 旋转；
3. 强光闪烁；
4. 长时间循环动画。

# 6. 按钮规范

## 6.1 按钮类型

| 类型 | 用途 | 样式 |
|---|---|---|
| Primary | 主要操作，如生成、保存、提交 | 靛蓝背景 |
| Secondary | 次要操作 | 白底边框 |
| Ghost | 辅助操作 | 透明背景 |
| Destructive | 删除、回滚危险操作 | 红色 |
| AI Action | AI 生成、AI 总结、自进化分析 | 渐变色 |
| Link | 文本跳转 | 主色文字 |

## 6.2 Primary Button

```tsx
className="rounded-xl bg-indigo-600 px-4 py-2 text-sm font-medium text-white shadow-sm transition hover:bg-indigo-700"
```

## 6.3 AI Action Button

```tsx
className="rounded-xl bg-gradient-to-r from-indigo-600 to-cyan-500 px-4 py-2 text-sm font-medium text-white shadow-sm transition hover:shadow-md"
```

使用场景：

1. 生成 Wiki；
2. AI 答疑发送；
3. 生成资源；
4. 生成诊断；
5. 自进化分析；
6. AI 总结页面。

## 6.4 按钮交互要求

1. 所有异步按钮必须有 loading 状态。
2. 危险操作必须二次确认。
3. AI 生成按钮必须显示“生成中”或“分析中”。
4. 保存类按钮成功后显示 Toast。
5. 禁止按钮无反馈。

# 7. 表单规范

## 7.1 表单视觉

表单应轻量、清晰，不使用传统后台密集表单。

推荐输入框：

```tsx
className="rounded-xl border border-slate-200 bg-white/80 px-3 py-2 text-sm shadow-sm transition focus:border-indigo-400 focus:ring-2 focus:ring-indigo-100"
```

## 7.2 表单布局

| 场景 | 布局 |
|---|---|
| 登录注册 | 居中卡片，左右分栏 |
| 资源生成 | 左侧配置，右侧预览 |
| Wiki 编辑 | 分块编辑 + 预览 |
| 管理配置 | 表单分组卡片 |
| 筛选表单 | 横向紧凑布局 |

## 7.3 表单状态

| 状态 | 要求 |
|---|---|
| 默认 | 白底，浅边框 |
| 聚焦 | 主色边框，淡色 ring |
| 错误 | 红色边框，错误文案 |
| 禁用 | 低透明度 |
| 加载 | 禁用输入并显示 loading |

## 7.4 表单文案

表单文案应具体，避免空泛。

示例：

```text
请输入你想生成的学习资源要求，例如：用初学者能理解的方式讲解递归调用栈。
```

不要写：

```text
请输入内容
```

# 8. 图表规范

## 8.1 图表总体要求

图表用于解释学习状态，不用于装饰。

要求：

1. 信息明确；
2. 标题清楚；
3. tooltip 有解释；
4. 配色克制；
5. 与卡片视觉统一；
6. 不使用 3D 图表；
7. 不使用花哨渐变柱。

## 8.2 推荐图表类型

| 页面 | 图表 |
|---|---|
| 学生首页 | 掌握度雷达图、学习趋势折线图 |
| 学习诊断 | 掌握度热力图、错因柱状图 |
| 自进化中心 | 策略效果折线图、更新次数统计 |
| 管理员 | LLM 调用量折线图、Agent 成功率图 |
| 学生画像 | 知识点掌握热力图、偏好标签图 |

## 8.3 Recharts 样式建议

1. 网格线使用 `#E2E8F0`，透明度降低；
2. 坐标轴使用 `#64748B`；
3. 主数据线使用 `#4F46E5`；
4. 成长数据使用 `#10B981`；
5. 风险数据使用 `#F59E0B` 或 `#EF4444`；
6. tooltip 使用圆角白底卡片。

## 8.4 禁止图表风格

禁止：

1. 3D 饼图；
2. 彩虹色柱状图；
3. 无标题图表；
4. 无单位图表；
5. 颜色无语义；
6. 密集到看不懂的图。

# 9. Wiki 页面视觉规范

## 9.1 页面定位

LLM Wiki 页面应像一个可持续成长的知识空间，不像普通 Markdown 文档。

## 9.2 推荐布局

```text
左侧：页面目录 / 知识点导航
中间：Wiki 正文内容
右侧：来源引用 / 关联知识点 / 掌握度 / 操作
底部：版本历史 / 错题 / 资源 / AI 建议
```

## 9.3 Wiki 正文规范

正文应按块呈现：

1. 一句话解释；
2. 前置知识；
3. 核心概念；
4. 关键代码或公式；
5. 典型例题；
6. 常见错误；
7. 关联知识点；
8. 我的笔记；
9. 来源引用。

每个块建议使用：

```tsx
className="rounded-2xl border border-slate-200/70 bg-white/80 p-5 shadow-sm"
```

## 9.4 来源引用视觉

来源引用必须清晰可信。

样式：

1. 使用小卡片；
2. 显示来源类型；
3. 显示资料名称；
4. 显示页码或片段；
5. 可点击展开原文片段。

来源标签示例：

| 来源类型 | 标签颜色 |
|---|---|
| 文档 | Indigo |
| 问答 | Cyan |
| 错题 | Red |
| 诊断 | Amber |
| 学生笔记 | Emerald |
| AI 生成 | Slate |

## 9.5 Wiki 版本历史

版本历史使用时间线展示：

```text
V1 AI 创建页面
V2 学生补充笔记
V3 AI 保存答疑内容
V4 回滚到 V2
```

每个版本必须展示：

1. 版本号；
2. 修改者；
3. 修改时间；
4. 修改摘要；
5. 查看按钮；
6. 回滚按钮。

# 10. 知识图谱视觉规范

## 10.1 图谱定位

知识图谱用于展示 Wiki 页面之间的关系，体现“知识空间”和“学习路径”。

## 10.2 节点规范

| 节点类型 | 视觉 |
|---|---|
| 已掌握知识点 | 翡翠绿色边框 |
| 一般知识点 | 靛蓝色边框 |
| 薄弱知识点 | 红色或橙色边框 |
| 当前知识点 | 主色填充或高亮 ring |
| 推荐下一步 | 青蓝色发光弱高亮 |
| 未学习知识点 | 灰色低透明度 |

## 10.3 边规范

| 关系类型 | 线条样式 |
|---|---|
| prerequisite | 实线箭头 |
| contains | 粗线 |
| confused_with | 虚线 |
| supports | 细实线 |
| next | 主色箭头 |
| resource_for | 点线 |

## 10.4 图谱交互

1. Hover 节点显示知识点摘要；
2. 点击节点打开详情抽屉；
3. 双击节点跳转 Wiki 页面；
4. 可按关系类型筛选；
5. 可按掌握度筛选；
6. 可高亮学习路径；
7. 支持重置视图。

## 10.5 禁止图谱风格

禁止：

1. 节点过多无筛选；
2. 边颜色随机；
3. 大面积发光；
4. 力导向图无限抖动；
5. 无图例；
6. 无掌握度语义。

# 11. Agent 状态展示规范

## 11.1 Agent 展示目标

Agent 展示不是装饰，而是证明系统不是简单套壳。

需要展示：

1. 哪个 Agent 被调用；
2. Agent 正在做什么；
3. 输入是什么；
4. 输出摘要是什么；
5. 是否成功；
6. 花了多久；
7. 是否调用大模型；
8. 是否更新 Wiki 或学生画像。

## 11.2 Agent 状态颜色

| 状态 | 样式 |
|---|---|
| pending | 灰色 |
| running | Indigo / Cyan，带轻微脉冲 |
| success | Emerald |
| warning | Amber |
| failed | Red |
| skipped | Slate |

## 11.3 Agent 调用链

推荐用时间线或步骤流：

```text
Orchestrator → Knowledge Agent → Wiki Agent → Review Agent → 保存结果
```

视觉要求：

1. 每个 Agent 是圆角节点；
2. 节点之间用细线连接；
3. 当前运行节点轻微高亮；
4. 成功节点显示对勾；
5. 失败节点显示错误原因；
6. 支持展开查看输入输出摘要。

## 11.4 Agent 卡片字段

每个 Agent 状态卡片包含：

1. Agent 名称；
2. 中文职责；
3. 状态；
4. 耗时；
5. 输出摘要；
6. 调用模型；
7. 查看详情按钮。

# 12. 学生画像展示规范

## 12.1 画像展示目标

学生画像页面需要让学生理解系统为什么这样判断自己，而不是只展示标签。

## 12.2 页面组成

1. 画像摘要卡片；
2. 学习目标；
3. 知识点掌握度；
4. 薄弱知识点；
5. 常见错误模式；
6. 学习偏好；
7. 解释风格；
8. 画像更新依据；
9. 画像版本。

## 12.3 画像标签规范

标签应带置信度或依据。

示例：

```text
偏好：分步骤讲解
依据：最近 5 次点赞中有 4 次来自分步骤解释
置信度：0.82
```

不要只显示：

```text
你喜欢分步骤讲解
```

## 12.4 掌握度展示

| 掌握度 | 颜色 | 文案 |
|---|---|---|
| 0-40 | Red | 明显薄弱 |
| 40-60 | Amber | 需要巩固 |
| 60-80 | Indigo | 基本掌握 |
| 80-100 | Emerald | 掌握较好 |

# 13. 自进化记录展示规范

## 13.1 展示目标

自进化页面要明确展示：

1. 系统发现了什么；
2. 为什么发现；
3. 更新了什么；
4. 影响哪些模块；
5. 是否有效；
6. 是否可以回滚。

## 13.2 时间线规范

自进化记录使用时间线：

```text
2026-05-20 发现递归调用栈薄弱
2026-05-20 更新答疑风格策略
2026-05-21 相关题目正确率提升
```

每个事件包含：

1. 事件标题；
2. 触发原因；
3. 证据；
4. 更新内容；
5. 风险等级；
6. 当前状态。

## 13.3 策略 Diff 规范

更新前后必须对比展示：

| 字段 | 更新前 | 更新后 |
|---|---|---|
| 回答长度 | long | medium |
| 是否加入代码示例 | false | true |
| 是否分步骤解释 | false | true |
| 是否加入易错提醒 | false | true |

## 13.4 风险等级

| 风险 | 视觉 |
|---|---|
| low | 绿色 |
| medium | 黄色 |
| high | 红色 |

## 13.5 回滚按钮

回滚按钮必须使用危险操作样式，并有确认弹窗。

确认文案示例：

```text
确定要回滚该策略吗？回滚后，后续答疑和推荐将恢复到上一个策略版本。
```

# 14. 学习路径时间线规范

## 14.1 设计目标

学习路径应像一个可执行的学习旅程，而不是普通列表。

## 14.2 节点视觉

| 状态 | 样式 |
|---|---|
| 未开始 | 灰色边框 |
| 进行中 | 主色高亮 |
| 已完成 | 绿色对勾 |
| 跳过 | 灰色低透明 |
| 建议复习 | Amber 提醒 |
| 薄弱补强 | Red/Amber 标签 |

## 14.3 节点内容

每个节点包含：

1. 知识点名称；
2. 任务类型；
3. 推荐理由；
4. 预计时间；
5. 当前掌握度；
6. 操作按钮：
   - 去学习；
   - 去练习；
   - 查看 Wiki；
   - 标记完成。

## 14.4 时间线方向

桌面端建议横向或纵向均可：

1. 复杂路径使用纵向；
2. 简单路径使用横向步骤条；
3. 移动端统一纵向。

# 15. AI 对话界面规范

## 15.1 设计目标

AI 答疑页面必须可信、专业、有依据。不能做成普通聊天娱乐界面。

## 15.2 聊天气泡

| 消息类型 | 样式 |
|---|---|
| 学生消息 | 右侧，淡靛蓝背景 |
| AI 消息 | 左侧，白色卡片 |
| 系统提示 | 居中，小号灰色 |
| Agent 状态 | 可折叠步骤条 |
| 来源引用 | AI 消息下方卡片 |

## 15.3 AI 回答结构

AI 回答建议包含：

1. 直接回答；
2. 分步骤解释；
3. 来源引用；
4. 关联知识点；
5. 下一步建议；
6. 保存到 Wiki 按钮。

## 15.4 输入框

输入框应支持：

1. 多行输入；
2. 发送按钮；
3. 课程上下文；
4. 知识点上下文；
5. Enter 发送 / Shift Enter 换行；
6. 生成中禁用重复发送。

## 15.5 流式输出

流式输出要求：

1. 文本逐步出现；
2. 显示“AI 正在结合 Wiki 与资料思考”；
3. 结束后展示引用；
4. 失败时显示重试按钮；
5. 不要显示夸张“思考中”动画。

# 16. 空状态设计

## 16.1 空状态原则

空状态不能只写“暂无数据”，必须告诉用户下一步能做什么。

## 16.2 空状态示例

### 没有 Wiki 页面

```text
还没有 Wiki 页面
上传课程资料后，系统可以自动整理出知识点页面。
按钮：上传资料 / 从资料生成 Wiki
```

### 没有学习记忆

```text
系统还没有形成长期学习记忆
当你完成问答、练习和反馈后，智能体会逐步理解你的学习偏好。
按钮：开始 AI 答疑
```

### 没有诊断报告

```text
还没有学习诊断
完成几道练习题后，系统可以分析你的薄弱知识点和下一步学习建议。
按钮：生成练习
```

### 没有自进化记录

```text
还没有策略进化记录
当系统发现稳定的学习偏好或薄弱模式后，会生成可解释的策略更新。
按钮：手动分析
```

## 16.3 空状态视觉

使用：

1. 轻量线性图标；
2. 柔和背景；
3. 明确按钮；
4. 不使用廉价机器人插画；
5. 不使用夸张表情包。

# 17. 加载状态设计

## 17.1 通用加载

使用 skeleton，不使用大面积转圈。

| 场景 | 加载方式 |
|---|---|
| 卡片数据 | Skeleton Card |
| 表格 | Skeleton Row |
| Wiki 页面 | Skeleton Paragraph |
| 图表 | Skeleton Chart |
| AI 生成 | Agent 状态 + 流式输出 |
| 文件解析 | 进度状态条 |

## 17.2 AI 任务加载

AI 任务必须显示过程感。

示例：

```text
正在检索课程资料...
正在读取相关 Wiki 页面...
正在结合你的学习画像...
正在生成个性化回答...
```

对应 Agent：

```text
Tutor Agent 运行中
Review Agent 校验中
Memory Agent 判断是否需要更新记忆
```

## 17.3 加载动效

允许：

1. 轻微 pulse；
2. skeleton shimmer；
3. 当前 Agent 节点高亮；
4. 进度条。

禁止：

1. 夸张加载动画；
2. 长时间无反馈；
3. 只显示“Loading...”；
4. 大面积转圈。

3. Tutor 使用左右辅助面板；

1. 侧边栏可折叠；
2. Wiki 页面右侧面板下移；

1. 底部导航代替侧边栏；
2. Wiki 页面单列；
3. 右侧信息变为折叠面板；
4. 表格改为卡片列表；
5. 图谱页面提供简化列表模式；
6. AI 输入框固定底部。

# 20. 禁止出现的低质量风格

以下风格禁止出现在智学工坊前端中。

## 20.1 禁止普通后台模板

不要出现：

1. 纯白背景 + 灰色表格 + 蓝色按钮；
2. 页面只有表格，没有学习空间感；
3. 所有模块长得像管理系统；
4. 左侧菜单拥挤且无视觉层级；
5. 页面缺少解释和推荐理由。

## 20.2 禁止廉价 AI 风格

不要出现：

1. 大机器人头像；
2. 霓虹蓝紫强发光；
3. “AI 魔法”式夸张动效；
4. 随机粒子背景；
5. 过度拟人化表情；
6. 夸张科技线框；
7. 大量“智能”“赋能”空话装饰。

## 20.3 禁止低质量布局

不要出现：

1. 卡片间距不统一；
2. 圆角不统一；
3. 图标风格混乱；
4. 颜色无语义；
5. 按钮大小不一致；
6. 页面层级混乱；
7. 内容没有留白；
8. 图表没有标题；
9. 表单没有错误提示；
10. 加载无反馈。

## 20.4 禁止不可信 AI 输出

不要出现：

1. AI 回答没有来源；
2. 推荐没有理由；
3. 诊断没有依据；
4. 自进化策略没有证据；
5. Wiki 页面没有版本；
6. 生成内容不能保存；
7. Agent 调用不可见。

# 21. 页面验收标准

前端页面完成后，应按以下标准验收。

## 21.1 视觉验收

| 验收项 | 标准 |
|---|---|
| 背景 | 有柔和层次，不花哨 |
| 卡片 | 圆角、阴影、间距统一 |
| 色彩 | 主色、状态色语义明确 |
| 字体 | 层级清楚 |
| 动效 | 轻微自然 |
| 图表 | 信息明确 |
| 移动端 | 可正常使用 |

## 21.2 功能验收

| 页面 | 必须体现 |
|---|---|
| Dashboard | 学习状态、推荐、自进化洞察 |
| Wiki | 页面、来源、关系、版本 |
| Tutor | 回答、引用、保存到 Wiki |
| Diagnosis | 薄弱点、错因、推荐动作 |
| Evolution | 策略版本、证据、回滚 |
| Agent | 调用链和状态 |

## 21.3 比赛展示验收

必须能清楚展示：

1. 资料变 Wiki；
2. Wiki 支撑答疑；
3. 问答能沉淀回 Wiki；
4. 错题触发诊断；
5. 诊断更新画像；
6. 自进化生成策略；
7. 推荐下一步学习；
8. Agent 协作过程可见。

# 22. 最终设计结论

智学工坊的 UI 应定位为：

> 具有高级教育科技感的个性化学习空间。

它不是普通后台，不是聊天机器人，也不是文档管理器，而是一个围绕学生成长的智能学习空间。

最终视觉目标：

```text
温暖但不幼稚
科技但不冰冷
高级但不复杂
智能但不神秘
结构化但不死板
```

所有页面和组件都应服务于四个核心展示：

1. 学习资料如何变成 LLM Wiki；
2. 学生画像如何持续形成；
3. 智能体如何协作完成学习任务；
4. 系统如何通过自进化越来越懂学生。
