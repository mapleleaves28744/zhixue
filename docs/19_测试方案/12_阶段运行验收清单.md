# 12_阶段运行验收清单

> 用途：约束每个开发阶段完成后必须在本地环境验证可运行，避免只完成代码但主链路断裂。

## 编写状态

- [x] 已补充

## 总原则

当前项目采用“本地优先，Docker 后置”的验收方式。除第21阶段 Docker 部署外，所有阶段默认使用本机 PostgreSQL、Redis、Python 虚拟环境和 Node.js 开发环境验证。

每个阶段完成时必须满足：

1. 本阶段新增或修改的代码能在本地环境启动或被测试覆盖；
2. 数据库结构变化必须执行 `python -m alembic upgrade head`；
3. 后端变化必须执行 `python -m pytest` 或至少执行相关测试文件；
4. 前端变化必须执行 `npm run typecheck` 和 `npm run build`；
5. 涉及 LLM 的变化必须在无真实 Key 时使用 Mock Provider 跑通；
6. 涉及真实 LLM Provider 的变化必须能 fallback 到 Mock；
7. 完成输出必须说明哪些检查已执行、哪些未执行以及原因。

## Stitch 前端联动验收

当前学生端采用“保留 Stitch 静态页 + 静态脚本接后端”的实现路线。除非任务明确说明并经人工确认要 React 组件化，否则所有阶段验收还必须检查：

1. 对应功能是否已接入 `frontend/public/stitch-pages/*.html` 中的现有页面；
2. 现有导航栏、侧边栏、顶部栏和主视觉布局是否保持不变；
3. 静态占位内容是否已替换为真实 API 数据、真实空状态或明确的 Mock 状态；
4. 页面是否通过 `frontend/public/stitch-pages/zhixue-static-api.js` 或对应静态页脚本调用 `/api/v1`；
5. 用户是否能在浏览器中完成该阶段最短操作闭环。

若后端 API 已完成但对应 Stitch 页面未接入，不得标记该阶段完成，只能标记为：

```text
后端实现完成，前端 Stitch 联动未完成。
```

## 本地基础环境

`.env` 默认连接本机服务：

```env
DATABASE_URL=postgresql+asyncpg://zhixue:zhixue_password@localhost:5432/zhixue
REDIS_URL=redis://localhost:6379/0
LLM_PROVIDER=mock
EMBEDDING_PROVIDER=mock
```

Docker 内部主机名 `postgres`、`redis` 只允许出现在 Docker 部署说明中，不作为普通阶段默认配置。

| 第7阶段 LLM Wiki 学习空间 | Wiki CRUD、版本、来源、关系、从资料生成 Wiki 可用；重复生成不破坏历史版本；`/knowledge` Stitch 页面可从资料生成 Wiki 并展示真实 Wiki/空状态。 |
| 第8阶段 大模型接口封装 | Mock Provider、OpenAI-compatible fallback、Prompt 渲染、LLM 日志测试通过；无 Key 可演示。 |

| 第12阶段 学习路径生成 | 学习路径模型/API/Planner Agent 可用；推荐理由可解释并关联画像、诊断或 Wiki；`/path-profile` Stitch 页面可展示真实路径与推荐理由。 |
| 第13阶段 学习资源生成 | 资源保存、生成、关联 Wiki 可用；内容有 citations 和 personalized_reason；`/assistant` 或 `/practice` Stitch 页面可生成并展示资源。 |

## 每阶段收尾输出要求

每阶段完成后，输出必须包含：

```text
本地运行结果：
- 后端检查：...
- 前端检查：...
- 数据库 migration：...
- LLM/Mock 检查：...
- 主链路检查：...
未执行项及原因：...
```

如果任一必要检查失败，不得宣称阶段完成，只能标记为“实现已提交，验收未通过”，并说明下一步修复项。
