## 当前测试策略

项目当前采用“本地优先，Docker 后置”的测试策略：

1. 日常开发优先验证本地 Python、Node、本机 PostgreSQL、本机 Redis 环境；
2. 后端任务运行 `python -m pytest` 或相关测试；
3. 前端任务运行 `npm run typecheck` 和 `npm run build`；
4. 数据库任务运行 `python -m alembic upgrade head`；
5. 涉及 LLM 的任务必须保证 Mock Provider 可运行；
6. Docker 只在第21阶段或部署专项阶段作为强制验收。
