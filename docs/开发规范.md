# 智学工坊开发规范

## 总则

- 每次只完成当前任务，不扩大重构范围。
- 前后端实现必须与 `docs/` 设计保持一致。
- 不提交 `.env`、API Key、真实用户隐私数据。
- 数据库结构变化必须通过 Alembic migration。
- 大模型调用必须通过统一 LLM Provider，缺少真实 Key 时使用 Mock Provider。

## 前端规范

- 使用 Next.js App Router、TypeScript、Tailwind CSS 和 shadcn/ui 源码组件。
- 页面只做组合和页面级状态，复杂请求放入 `frontend/services/`。
- 通用组件放入 `frontend/components/`，类型放入 `frontend/types/`。
- 避免 `any`，API 响应必须使用泛型类型。
- 视觉遵守 Luminous Warmth：暖白背景、金棕主色、暖橙强调、玻璃拟态卡片、轻阴影和清晰层级。
- 业务页面不得散落裸 `fetch`，统一使用 `frontend/lib/request.ts`。

## 后端规范

- API Router 只负责参数接收、依赖注入、调用 Service 和返回响应。
- Service 层负责业务流程、权限上下文、事务和 Agent/LLM 调用。
- Repository 层只处理数据库查询和持久化，不写业务决策。
- Python 代码必须使用类型注解。
- 业务异常使用统一异常体系，不抛裸字符串异常。

## API 与响应

- 业务接口统一挂载 `/api/v1`。
- 成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "request_id": "req_xxx"
}
```

- 错误响应必须包含 `code`、`message`、`detail`、`request_id`。
- 涉及 `user_id` 和 `course_id` 的查询必须校验当前用户权限。

## 数据库规范

- 表名使用 snake_case 复数，如 `users`、`courses`。
- SQLAlchemy Model、Pydantic Schema 和 migration 必须同步。
- pgvector 检索必须带 `course_id` 过滤，避免跨课程或跨学生泄露。

## Agent、Wiki 与自进化

- Agent 类继承 `BaseAgent`，输入输出使用 `AgentContext` 和 `AgentResult`。
- Agent 不允许自动修改源代码、数据库结构、权限规则或部署配置。
- Wiki 内容应保留来源、版本和关系。
- 自进化策略必须可解释、可记录、可版本化、可回滚、有证据和风险等级。

## Prompt 规范

- Prompt 不散落在业务代码中，优先放入 `backend/app/prompts/` 或 `prompt_versions`。
- Prompt 按场景命名，如 `tutor.qa`、`wiki.generate`、`resource.generate`、`evolution.analyze`。

## 测试与提交

- 前端任务至少运行 `npm run typecheck` 和 `npm run build`。
- 后端任务至少运行相关 pytest 或启动 FastAPI 检查。
- 数据库任务运行 `alembic upgrade head`。
- 输出中必须说明修改文件、数据库变更、API 变更、测试结果和风险。
