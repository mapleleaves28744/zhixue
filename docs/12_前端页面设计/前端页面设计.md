# 智学工坊：前端页面设计

文档版本：V1.0  
适用位置：`docs/前端页面设计.md`  
技术栈：Next.js App Router + TypeScript + Tailwind CSS + shadcn/ui + Framer Motion + Recharts  
项目名称：智学工坊  
核心特色：自进化学习智能体、LLM Wiki 学习空间、个性化资源生成、多智能体学习闭环

---

# 1. 前端总体设计目标

智学工坊前端不是普通后台管理系统，而是一个面向高校学生的“个人学习空间”。页面需要体现：

1. 学习空间感；
2. 知识组织感；
3. AI 辅助但不过度廉价化；
4. 自进化过程可解释；
5. 多智能体协作可展示；
6. 课程、Wiki、问答、资源、诊断、推荐形成闭环。

前端核心体验主线：

```text
学生进入首页
  → 查看学习状态
  → 进入 LLM Wiki 学习空间
  → 向 AI Tutor 提问
  → 生成个性化学习资源
  → 完成练习与错题复盘
  → 查看学习诊断
  → 触发自进化策略更新
  → 系统推荐下一步学习路径
```

# 2. 用户角色与前端范围

## 2.1 游客

游客只能访问公开页面：

1. 首页 `/`
2. 登录页 `/login`
3. 注册页 `/register`

## 2.2 学生

学生是系统核心用户，拥有完整学习端功能。

学生访问：

1. 学生首页；
2. 学生画像；
3. 学习记忆；
4. LLM Wiki；
5. 知识图谱；
6. 学习路径；
7. 个性化资源；
8. AI 答疑；
9. 练习题；
10. 错题本；
11. 学习诊断；
12. 推荐；
13. 自进化中心。

## 2.3 教师

当前版本不建设完整教师端。教师角色建议作为“课程内容维护者”预留，可在后续扩展以下能力：

1. 上传课程资料；
2. 维护公共课程模板；
3. 查看课程知识结构；
4. 管理课程级初始知识库。

MVP 阶段不单独提供 `/teacher` 路由，避免偏离学生个性化学习主线。

## 2.4 管理员

管理员负责系统维护和比赛演示数据管理。

管理员访问：

1. 管理员首页；
2. 用户管理；
3. Agent 运行日志；
4. 大模型调用日志；
5. 自进化策略管理；
6. 系统配置。

## 12.3 fetch 封装建议

```ts
export async function request<T>(
  url: string,
  options: RequestInit = {}
): Promise<T> {
  const token = getToken()

  const res = await fetch(`${process.env.NEXT_PUBLIC_API_BASE_URL}${url}`, {
    ...options,
    headers: {
      "Content-Type": "application/json",
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
      ...options.headers,
    },
  })

  const json = await res.json()

  if (json.code !== 0) {
    throw new Error(json.message || "请求失败")
  }

  return json.data as T
}
```

# 13. UI 风格规范

## 13.1 视觉关键词

1. 高级教育科技感；
2. 柔和渐变；

9. 避免廉价 AI 风格；
10. 避免普通后台模板。

## 13.2 色彩规范

建议采用蓝紫青绿色系，强调知识、智能、成长。

| 内边距 | `p-5` 或 `p-6` |

## 13.5 动效规范

使用 Framer Motion：

| 场景 | 动效 |
|---|---|
| 页面进入 | opacity + y: 12 |
| 卡片出现 | stagger children |
| 按钮 hover | scale 1.02 |

| Agent 调用链 | 节点逐步高亮 |

4. 关联知识点用胶囊标签；

# 18. 最终前端定位

智学工坊前端应定位为：

> 一个面向高校学生的智能学习空间，而不是普通后台系统。

核心页面要围绕：

```text
Dashboard：看状态
Wiki：沉淀知识
Tutor：即时答疑
Resources：生成资源
Diagnosis：发现问题
Evolution：展示成长
Recommendations：给出下一步
```

前端最重要的展示效果是：

1. 学习资料变成 Wiki；
2. 问答结果能保存到 Wiki；
3. 错题能关联知识点；
4. 诊断能发现薄弱点；
5. 自进化能解释系统为什么改变策略；
6. 推荐能说明为什么下一步学这个。

只要这条链路在页面上清晰，项目就不会像普通 AI 套壳，而会呈现为一个完整的个性化学习智能体系统。
