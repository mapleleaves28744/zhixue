## 2. 导航映射

当前所有 Stitch 页面侧边导航统一为 7 个入口：

| 导航文案 | 图标 | 路由 |
|---|---|---|
| 首页 | `home` | `/` |
| 课程空间 | `school` / `grid_view` | `/courses` |
| AI 学习助手 | `smart_toy` | `/assistant` |
| 课程知识库 | `menu_book` | `/knowledge` |
| 练习与诊断 | `assignment_turned_in` / `inventory_2` | `/practice` |
| 学习仪表盘 | `dashboard` | `/dashboard` |
| 学习路径与个人模型 | `route` | `/path-profile` |

静态 HTML 在 iframe 内运行，所有跨页面跳转必须使用：

```html
target="_parent"
```

否则链接只会在 iframe 内部跳转，导致路由状态和浏览器地址栏不一致。

## 3. 当前权限策略

当前为演示原型阶段，暂不做前端路由鉴权。

后续接入 Auth 后建议：

1. `/login`、`/register` 作为公开路由。
2. 当前 7 个主路由作为学生登录后路由。
3. 未登录访问学生路由时跳转 `/login`。
4. 登录后根据用户角色进入 `/`。
5. 教师端和管理员端只在后续阶段补充，不进入当前学生主链路。

## 4. 后续路由演进

组件化后可以保持当前 7 个主路由不变，在页面内部增加局部 tab 或 query 参数，不建议再拆出大量顶层页面。

推荐：

```text
/courses        课程资料 / 课程 Wiki / 入口卡片
/assistant      Tutor 问答 / Agent 状态 / 引用来源

/practice       练习 / 答题 / 错题 / 诊断
/dashboard      学习状态 / 任务 / 趋势 / 提醒
/path-profile   学习路径 / 画像 / 自进化策略
```
