# 01_前端页面总览

> 当前版本说明：本文件以 `frontend/` 当前实现为准。现阶段前端采用 Next.js App Router 承载 Stitch 导出的 Luminous Warmth 静态页面，后续再逐页拆分为 React 组件并接入后端 API。

> 当前范围冻结：重新推进到第20阶段时不再新增教师端和管理员端页面，不创建 `/teacher/*` 或新的 `/admin/*` React 页面。前端工作优先围绕 7 个学生端 Stitch 主入口接入真实 API、空状态、错误提示和验收测试。

## 1. 当前前端形态

当前前端是“可演示原型 + 后续组件化重构基础”：

1. Next.js 负责路由、页面承载、全局字体与基础 shell。
2. `frontend/public/stitch-pages/*.html` 保存 Stitch 导出的视觉页面。
3. `frontend/components/StitchFrame.tsx` 使用全屏 iframe 承载静态页面，保证 Stitch 视觉不被 Next 外层样式破坏。
4. 7 个主导航入口已经统一：主页、课程空间、AI 学习助手、课程知识库、练习与诊断、学习仪表盘、学习路径与个人模型。
5. 后续接入真实 API 时，优先维护 7 个学生端 Stitch 主入口，不扩展教师端或管理员端新页面。

## 3. 页面聚合原则

当前不再拆成十几个独立小页面，而是按用户实际使用的大模块聚合：

1. 课程相关入口合并到“我的课程空间”。
2. Wiki、知识点、知识关系合并到“课程知识库”。
3. Tutor、问答、Agent 状态合并到“AI 学习助手”。
4. 题库、答题、错题、诊断合并到“练习与诊断”。
5. 画像、路径、自进化策略合并到“学习路径与个人模型”。
6. 学习状态概览保留为“学习仪表盘”。
7. `/` 作为总入口首页，不再映射到学习仪表盘。

## 4. 后续组件化方向

后续重构时应保持视觉不变，按以下顺序逐步替换 iframe 静态页：

1. 先抽取全局 AppShell、侧边导航、玻璃卡片、按钮、状态标签。
2. 再拆每个页面的业务区块组件。
3. 最后接入 `frontend/services/*` 的 API 请求与真实状态管理。

重构期间必须保留 Stitch 页面作为视觉基准，不允许为了组件化牺牲当前外观、颜色、布局和密度。
