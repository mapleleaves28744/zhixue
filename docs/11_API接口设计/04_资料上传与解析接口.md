# 04_资料上传与解析接口

> 用途：文件上传、解析状态、切片结果、来源管理接口。

## 编写状态

- [x] 已补充上传、列表、详情、解析状态和同步解析接口

## 接口清单

| 功能 | 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|---|
| 上传资料 | POST | `/api/v1/materials/upload` | Course Owner/Admin | 支持 PDF、DOCX、Markdown、TXT |
| 资料列表 | GET | `/api/v1/materials?course_id=...` | Course Owner/Admin | 按课程查询资料 |
| 资料详情 | GET | `/api/v1/materials/{material_id}` | Course Owner/Admin | 返回资料元数据 |
| 解析状态 | GET | `/api/v1/materials/{material_id}/parse-status` | Course Owner/Admin | 返回解析状态、错误和文本信息 |
| 解析资料 | POST | `/api/v1/materials/{material_id}/parse` | Course Owner/Admin | 同步提取文本，暂不做 OCR 和切片 |

## 上传参数

`POST /api/v1/materials/upload` 使用 `multipart/form-data`：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `course_id` | uuid | 是 | 课程 ID |
| `file` | file | 是 | 课程资料文件 |

约束：

- 允许扩展名：`pdf`、`docx`、`md`、`txt`。
- 最大文件大小：50MB。
- 文件保存到本地 `storage/materials/<course_id>/<material_id>.<ext>`。
- 初始 `parse_status=pending`。

## 解析行为

解析资料时：

1. 先将状态更新为 `processing`。
2. TXT/Markdown 直接按 UTF-8 读取。
3. DOCX 使用 `python-docx` 提取段落文本。
4. PDF 使用 `pypdf` 提取页面文本，暂不做 OCR。
5. 成功后写入 `text_hash`，并在 `extra_meta` 中记录 `parsed_text_path` 和 `text_length`。
6. 失败后写入 `parse_status=failed` 和 `parse_error`。

所有接口使用统一响应结构：`code`、`message`、`data`、`request_id`。
